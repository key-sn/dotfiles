# dotfiles プロジェクトルール

このリポジトリ（dotfiles）の管理・実装を行う際は、以下のルールを常に遵守してください。

1. **冪等性（Idempotency）の担保**
   - シェルスクリプトや設定ファイルに追記を行う際は、複数回実行されても問題ないように記述すること。
   - 例: 単純な `echo` ではなく、既に設定が存在するか確認してから追記する。

2. **OSの互換性**
   - 新しい設定やスクリプトを追加する場合、macOS と Linux の両方で動作するか、もしくはOS判定の分岐を入れること。

3. **シンボリックリンクの原則**
   - 設定ファイルは直接 `~/.config/` などに作成せず、必ず `~/dotfiles/symlinks/` 内に実体を置き、そこから所定の場所へシンボリックリンクが張られていることを前提として作業すること。
